# 기억 가능한 대화량
llama3.1:8b의 컨텍스트 윈도우가 128K 토큰이지만 실질적인 한계는 훨씬 작습니다.
현재 프롬프트 구성에서 토큰 사용량을 계산하면:
---
```
항목                  토큰(대략)
시스템 프롬프트        ~200RAG 
컨텍스트 (청크 3개)    ~1,500
DB 결과                ~300
답변 생성 여유분       ~1,000
고정 사용량            ~3,000
```
한 턴(질문+답변)이 평균 500토큰이라 가정하면 10~15턴 정도가 안정적인 기억 한계입니다. 
그 이상 넣으면 응답 속도가 급격히 느려지고 LLM이 앞 내용을 무시하기 시작합니다.

---

#  구현 방식 선택지
## 방식 1 — 전체 기록 전달 (단순)
모든 이전 대화를 그대로 넣습니다. 구현이 쉽지만 대화가 길어지면 느려집니다. 최근 N턴만 자르는 슬라이딩 윈도우를 함께 적용하면 됩니다.
## 방식 2 — 요약 메모리 (LangChain ConversationSummaryMemory)
오래된 대화는 LLM이 요약해서 압축 저장합니다. 더 많은 대화를 기억할 수 있지만 요약 LLM 호출이 추가로 발생합니다.
## 방식 3 — 벡터 메모리 (Qdrant 활용)
이전 대화를 Qdrant에 임베딩해두고 현재 질문과 유사한 과거 대화만 검색해서 주입합니다. 사실상 무제한이지만 구현 복잡도가 높습니다.
